## 三、技术方案选型

### 3.1 ASR (语音→文本) — **sherpa-onnx** ✅ 推荐

| 方案 | 免费 | 离线 | 中文 | 流式 | 延迟 | 备注 |
|------|------|------|------|------|------|------|
| **sherpa-onnx** | ✅ | ✅ | ✅ | ✅ | **~200-500ms** | 轻量，C++核心，支持关键词检测 |
| Vosk | ✅ | ✅ | ✅ | ✅ | ~300-800ms | 备选，模型较大 |
| faster-whisper | ✅ | ✅ | ✅ | ❌ | ~1-3s | 精度最高但非流式，需 torch |
| 百度/讯飞 | 有免费额度 | ❌ | ✅ | ✅ | ~500ms+网络 | 需联网+注册 |

**选择 sherpa-onnx 理由**:
- 原生支持 **流式识别** (Streaming ASR)，说话过程中就能出结果
- 内置 **关键词检测** (Keyword Spotting) 可用于唤醒词
- C++ 引擎极轻量，CPU 即可实时，GPU 可进一步加速
- 有预训练中文模型 (SenseVoice, Paraformer, Zipformer 等)
- Python 3.14 有 wheel

### 3.2 TTS (文本→语音) — **edge-tts** ✅ 推荐

| 方案 | 免费 | 离线 | 中文 | 质量 | 延迟 |
|------|------|------|------|------|------|
| **edge-tts** | ✅ | ❌(需网络) | ✅ | ⭐⭐⭐⭐⭐ | ~0.5-1.5s |
| pyttsx3 | ✅ | ✅ | ✅ | ⭐⭐ | ~100ms |
| sherpa-onnx TTS | ✅ | ✅ | ✅ | ⭐⭐⭐ | ~200ms |
| Coqui TTS | ✅ | ✅ | ⚠ | ⭐⭐⭐⭐ | ~1-2s |

**选择 edge-tts 理由**:
- 已安装，微软 Edge 云端 TTS 引擎，完全免费无限量
- 中文语音质量极高，多种音色可选 (晓晓、云希 等)
- 支持流式输出，可边生成边播放
- 唯一缺点是需要网络 (本机已联网)
- **备选**: sherpa-onnx 也自带离线 TTS，可作为断网回退

### 3.3 Pi Agent 集成 — **RPC 模式** ✅

Pi SDK 提供两种集成方式:
1. **Node.js SDK** — TypeScript/JS 原生，功能最全
2. **RPC 模式** — `pi --mode rpc` 通过 stdin/stdout JSON 协议通信

**选择 RPC 模式理由**:
- Python 主程序可直接通过 subprocess 与 pi 通信
- JSON-line 协议简单可靠
- 支持所有核心功能: prompt, steer, abort, get_state 等
- 无需 Node.js 编程，整个项目用 Python 统一

### 3.4 音频 I/O — **sounddevice** ✅

- 已安装，跨平台 PortAudio 封装
- 直接通过设备 ID 指定蓝牙耳机输入/输出
- 支持回调模式，低延迟实时音频流

## 五、延迟分析

| 环节 | 预估延迟 | 说明 |
|------|----------|------|
| 蓝牙音频传输 | ~40-120ms | HFP 编码+传输固有延迟 |
| ASR 流式识别 | ~200-500ms | 说完后到出文字 |
| Pi Agent 响应 | ~1-5s | LLM 推理时间 (取决于模型和问题复杂度) |
| TTS 生成 | ~0.5-1.5s | edge-tts 网络延迟+生成 |
| 音频回传播放 | ~40-120ms | 蓝牙传输 |
| **总端到端** | **~2-7s** | 从说完话到开始听到回复 |

> 这个延迟对于语音助手交互是可接受的 (类似 Siri/小爱同学 的体验)。
> 如果启用 TTS 流式播放 (边生成边播)，首字节延迟可降至 ~2-3s。

## 六、唤醒词方案

由于 "Hi,派" 和 "休息，派" 是中文混合短语，纯关键词检测引擎可能不够准确。
**最佳方案**: 使用 sherpa-onnx 的 **流式 ASR 持续识别**，在识别结果中做关键词匹配:

- ASR 持续运行，低功耗监听
- 在文本结果中匹配: `嗨派`、`hi派`、`嘿派` → 唤醒
- 匹配: `休息派`、`休息吧派` → 休眠
- 活跃状态下匹配: `派，`/`派,` 开头 → 提取后续内容发送给 Pi

这比硬件唤醒词引擎更灵活，支持模糊匹配和中英混合。
